Страница регистрации волонтера

Волонтер не регистрируется сразу, он просто подает заявку на регистрацию. Админ проверит все предоставленные данные, рассмотрит заявку и примет решение о регистрации

Что примерно может получиться:

У страницы должен быть роут “/register/volunteer”

На странице должны быть следующие компоненты:

  1. Кнопка-ссылка обратно на главную страницу. При нажатии перекидывает пользователя на главную страницу сайта
  1. Компонент Tabs (есть библиотечный) для переключения между регистрацией обычного пользователя и волонтера
  1. Блок с текстом, чтобы волонтер понимал, как будет проходить процесс регистрации, где будет описано, почему волонтер подает заявку и почему важно предоставить как можно больше информации
  1. Форма для регистрации волонтера:
  • Инпут для имени. Обязательное поле. При вводе происходит мгновенная валидация, должен быть минимум один символ, не считая пробелов
  • Инпут для фамилии. Обязательное поле. При вводе происходит мгновенная валидация, должен быть минимум один символ, не считая пробелов
  • Инпут для отчества. Необязательное поле.
  • Инпут для почты. Обязательное поле. При вводе происходит мгновенная валидация почты
  • Инпут для информации о себе. Обязательное поле. При вводе происходит мгновенная валидация, должно быть минимум 10 символов, не считая пробелов
  • Инпут для опыта волонтерства. Обязательное поле. Можно вводить только числа от 1 до 100
  • Инпут для количества животных нашедших дом. Необязательное поле. Можно вводить только числа от 0 до 10000
  • Инпут для соцсетей. Может выглядеть примерно как на ютубе
  • Кнопка “Зарегистрироваться”. При нажатии, если все данные верные, то происходит отправка заявки, выводится окно с информацией, что заявка отправлена и происходит переход на главную страницу сайта. Если данные неверные, то выделяются неверно заполненные поля и под каждым таким полем пишется конкретная ошибка. Такая валидация возможна с помощью React Hook Form. Если ошибка при отправке формы прилетает уже с сервера, то нужно вывести всплывающее окно с ошибкой. Для такого функционала хорошо подходит библиотечный компонент Toast
  • Надпись с ссылкой “Уже зарегистрированы? Вход”. При нажатии на ссылку происходит переход на страницу входа
